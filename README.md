# Curly

Curly is a self-hosted media control panel for personal movie and anime libraries. It combines a custom Next.js streaming UI, Jellyfin media delivery, Filebrowser-backed file management, and a hardened local auth layer with admin TOTP.

## Features

- Custom home, watch, file, and admin routes on one domain
- Local `admin` and `viewer` roles stored in PostgreSQL
- Argon2id password hashing, HTTP-only sessions, and admin TOTP
- Audit trail for logins, disables, password resets, and library refreshes
- Admin-only file manager on the same media root Jellyfin scans
- Terminal commands for user creation, password resets, and disabling accounts
- Ubuntu 24.04 install flow with Docker Compose, nginx, and Certbot

## Stack

- Next.js 15 + TypeScript
- Prisma + PostgreSQL
- Jellyfin
- Filebrowser
- Docker Compose
- Host `nginx` and Certbot

## Quick Install

Clone the repo onto an Ubuntu 24.04 server, point your DNS record at that server, then run:

```bash
git clone https://github.com/Matei02355/Curly.git
cd Curly
sudo ./install.sh \
  --domain media.example.com \
  --email admin@example.com \
  --media-root /srv/curly/media \
  --admin-user admin
```

Missing flags fall back to prompts. The installer will:

1. Verify Ubuntu 24.04, root access, and DNS alignment.
2. Install Docker, Compose, Node.js, nginx, Certbot, UFW, and PostgreSQL client tools.
3. Create the media folders under the chosen media root.
4. Write `.env` values for Curly, PostgreSQL, Jellyfin, and Filebrowser.
5. Install Node dependencies and generate the Prisma client.
6. Start PostgreSQL, Jellyfin, and Filebrowser.
7. Apply the initial Prisma migration and create the first Curly admin user.
8. Initialize Jellyfin with a service account, create `Movies`, `Anime`, and `Shows` libraries, and write the resulting API key back to `.env`.
9. Build and start the Curly app container.
10. Configure nginx, request the Let's Encrypt certificate, enable HTTPS redirect, and dry-run renewal.

## Media Layout

Curly assumes one local media root. The default layout is:

```text
/srv/curly/media
├── anime
├── movies
├── shows
└── uploads
```

The same host path is mounted into Jellyfin and Filebrowser.

## User Management

Create a viewer:

```bash
npm run user:create -- --role viewer --username demo --password 'strong-password-here'
```

Create an admin:

```bash
npm run user:create -- --role admin --username ops --password 'another-strong-password'
```

Reset a password:

```bash
npm run user:reset-password -- --username demo --password 'new-strong-password'
```

Disable a user:

```bash
npm run user:disable -- --username demo
```

## Manual Setup

If you are not using `install.sh`, use this order:

```bash
cd /path/to/Curly
cp .env.example .env
npm ci
npm run prisma:generate
docker compose up -d postgres jellyfin filebrowser
npm run prisma:migrate:deploy
npm run bootstrap-admin -- --username admin --password 'change-this-now'
node scripts/jellyfin-init.mjs
docker compose up -d --build app
```

Then put nginx in front of `127.0.0.1:3000` with `ops/nginx/curly.conf.template` and run Certbot for your domain.

## Environment

Start from `.env.example`. Important keys:

- `APP_URL`: public Curly URL
- `DATABASE_URL`: host-side PostgreSQL URL for CLI scripts
- `DATABASE_URL_DOCKER`: container-side PostgreSQL URL for the app
- `MEDIA_ROOT`: host media root
- `JELLYFIN_API_KEY`: generated by `scripts/jellyfin-init.mjs`
- `JELLYFIN_USER_ID`: generated by `scripts/jellyfin-init.mjs`
- `FILEBROWSER_PROXY_PATH`: default `/api/files`

## Operations

Rebuild and restart:

```bash
docker compose up -d --build
```

Follow logs:

```bash
docker compose logs -f app
docker compose logs -f jellyfin
docker compose logs -f filebrowser
```

Dry-run certificate renewal:

```bash
sudo certbot renew --dry-run
```
